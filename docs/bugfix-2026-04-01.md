# Bug Fix Session — 2026-04-01

## BUG-01: Backend process dies immediately on `start-local.sh`

**Symptom:** `./start-local.sh` printed `BUILD SUCCESSFUL in 2s` then `ERROR: Backend process died.`  
**Root cause:** Spring Boot 4.0.x + GraalVM native plugin marks `:bootRun` as `UP-TO-DATE` when all upstream AOT tasks are up-to-date. Gradle exits immediately without launching the JVM.  
**Fix:** `backend/build.gradle` — added `outputs.upToDateWhen { false }` to the `bootRun` task.

---

## BUG-02: All admin sub-page navigations redirect to dashboard

**Symptom:** Clicking any card on `/admin` (datasources, epr-config, audit-search) immediately redirected to `/dashboard` or `/`.  
**Root cause:** All three sub-pages guarded access with `identityStore.user?.role !== 'SME_ADMIN'`. The `identityStore` requires `setToken()` with a JS-readable JWT, but the app switched to HttpOnly cookies (Story 5.4+). `identityStore.user` is always `null` → role check fails → redirect fires on every visit.  
**Fix:** Replaced `useIdentityStore()` with `useAuthStore()` (populated from `/me` API response) in `datasources.vue`, `epr-config.vue`, and `audit-search.vue`. Added explicit `import { useAuthStore } from '~/stores/auth'` to each page so Vitest module mocking works correctly.

## BUG-03: Admin nav link never visible in sidebar

**Symptom:** The Admin section never appeared in the sidebar navigation, even for `SME_ADMIN` users.  
**Root cause:** `AppSidebar.vue` computed `isAdmin` as `role.value === 'ADMIN'`, but the actual role value from the backend is `'SME_ADMIN'`.  
**Fix:** `AppSidebar.vue:131` — changed comparison to `role.value === 'SME_ADMIN'`.

## BUG-04: EPR config JSON editor never rendered

**Symptom:** The `/admin/epr-config` page showed the header and buttons but no editor between them.  
**Root cause (primary):** `epr-config.vue` referenced the component as `<MonacoEditor>`, but Nuxt registers components in subdirectories with their folder as a prefix: `components/Admin/MonacoEditor.vue` → `AdminMonacoEditor`. Vue silently renders nothing for unresolved component names.  
**Root cause (secondary, would have blocked even after renaming):** The Monaco editor itself never rendered due to Vite worker-bundling incompatibility (`new URL('…editor.worker', import.meta.url)` fails when `monaco-editor` is excluded from `optimizeDeps`), missing Monaco CSS import, and a race condition between the async `import('monaco-editor')` and `loadConfig()` resolving first.  
**Fix:**
- `epr-config.vue`: `<MonacoEditor>` → `<AdminMonacoEditor>`
- `MonacoEditor.vue`: replaced Monaco entirely with a `<textarea>` (monospace, resizable, `h-96`). Backend validation makes the Monaco JSON language server redundant, and the textarea is unconditionally reliable.
- Removed `exclude: ['monaco-editor']` from `nuxt.config.ts` `optimizeDeps`.

## FEAT-01: JSON format toggle on EPR config editor

**Request:** Add a button to the JSON textarea that pretty-prints the content for easier manual editing.  
**Implementation:** `MonacoEditor.vue` — added a `{ } Format` button above the textarea.
- Clicking formats the content via `JSON.stringify(JSON.parse(value), null, 2)` and activates an indigo highlight on the button.
- Clicking again deactivates the indicator (content stays formatted).
- Any manual keystroke in the textarea resets the indicator automatically.
- If the JSON is malformed at the time of formatting, an inline error message is shown and the content is left untouched.
